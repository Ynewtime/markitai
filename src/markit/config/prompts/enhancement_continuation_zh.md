请按照以下规则优化 Markdown 文档片段的格式。

【重要】这是文档的续段（非首段），请注意：
1. **不要**生成 YAML Frontmatter 块（`---` ... `---`）
2. 在输出末尾以 HTML 注释形式输出本段新发现的实体和主题

## 核心原则：尊重原文

**严禁修改原文档中的任何实际内容**，包括但不限于：
- **表格列头/表头**: 必须保持原文，禁止翻译或改写
- **正文文本**: 所有段落、句子必须保持原样
- **数据内容**: 表格数据、列表内容必须原样保留
- **专有名词**: 人名、地名、产品名等保持原文

## 清理规则

### 1. 垃圾内容清理
- **页眉页脚**: 移除文档顶部/底部重复出现的公司名称、日期、机密标识等
- **水印干扰**: 移除扫描件中的斜体水印文字
- **图表残留清理**: 移除从图表/图片中错误提取的内容：
  - 连续的独立数字行（如 "12"、"10"、"8"...通常是Y轴刻度）
  - 图表轴标签（如 "Row 1"、"Column 1"、"Category A"）
  - 图例文字（如 "Series 1"、"数据系列1"）
- **页码清理**: 移除独立成行的页码
- **乱码字符**: 移除OCR识别产生的乱码

### 2. PPT/演示文稿特殊处理
对于 PPT/PPTX 转换的内容，必须遵循标题层级：
- 每张幻灯片标题 → `##` (h2)，如 `## 第 N 页` 或 `## Slide N`
- 幻灯片内的一级标题 → `###` (h3)
- 幻灯片内的二级标题 → `####` (h4)

### 3. 空行规范化
- 标题上方保留一个空行
- 标题下方内容前保留一个空行
- 段落之间保留一个空行
- 移除连续超过两个的空行

### 4. GFM 规范
- 无序列表使用 `-` 作为标记符
- 代码块使用 ``` 并标注语言标识符
- **代码块内容必须完整保留，不得删除或修改**
- 表格对齐规范化

### 5. 元数据抽取
从本段内容中抽取新发现的实体和主题，**在输出末尾**以如下格式输出：

```
<!-- PARTIAL_METADATA: {{"entities": ["实体1", "实体2"], "topics": ["主题1"]}} -->
```

注意：
- entities 为关键实体（人名、组织、产品、技术术语等）- 保持原文
- topics 为主题标签 - 可使用中文
- 如果没有新实体或主题，对应数组留空
